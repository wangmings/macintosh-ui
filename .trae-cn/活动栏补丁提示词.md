# 活动栏补丁提示词（顺序化执行版）

## 角色
你是源码补丁助手。目标是在 `workbench/workbench.desktop.main.js.txt` 里实现最小化注入改动。

## 补丁功能
1. 让 `workbench.activityBar.location` 真正支持 `top / bottom / hidden / default`
2. 当配置为 `top/bottom` 时，活动栏走侧边栏内嵌创建路径（不走原独立栏位创建路径）
3. 当配置从 SOLO 恢复到 IDE 时，activitybar 显隐状态按运行时值恢复，避免错误强制显示
4. 修复底层配置读取被硬编码覆盖的问题，确保读取到真实配置值

## 核心原则
1. 只改“新增代码”和“实现该功能必须改动的代码”
2. 最小改动，禁止画蛇添足
3. 每个新增/修改关键行上方加 `// 新增:` 注释
4. 禁止新增调试代码
5. 不新增包装函数，不重构未改动逻辑

---

## 执行顺序（必须按顺序）

### 第 1 步：修配置读取根因（底层）
目的：禁止底层硬编码覆盖配置值。

实现点：配置服务 `getValue(i, e)` 的 `switch(t)`。

必须满足：
- 删除 `case "workbench.activityBar.location": return "default";`
- 保持回落：`return this.m.getValue(t, s)`

---

### 第 2 步：配置项扩展
目的：支持 `top/bottom/hidden/default`。

实现点：`workbench.activityBar.location` 配置定义。

必须满足：
- 把 `enum: ["default"]` 改为 `enum: ["top", "bottom", "hidden", "default"]`

---

### 第 3 步：侧边栏嵌入分支（create）
目的：`top/bottom` 时直接走侧边栏内嵌逻辑。

实现点：`case "workbench.parts.sidebar"` 的 `create(e)`。

必须满足：
1. 保留原有首行判断不变：
- `if (this.C.isSoloMode) return this.g.create(e);`
2. 在其后读取配置：
- `const t = this.q.Yc.getValue("workbench.activityBar.location");`
3. 增加 top/bottom 早返回：
- `if (t === "top" || t === "bottom") return this.g.create(e);`
4. 将原本直接创建的三个容器按原语义继续执行（只是变量顺序后移）：
- 先创建 builtin 容器并执行 `this.f?.create(...)`
- 再创建 `icube-sidebar-separator`
- 最后创建 `third-party` 容器并 `this.g.create(...)`
5. 保持返回值与结构不变：
- 仍然返回 `e`
6. 禁止新增包装函数或额外分支，`create(e)` 只做上述最小改动

---

### 第 4 步：活动项显示数量控制
目的：保持可见项计算与注入行为一致。

实现点：composite bar 可见项计算函数。

必须满足：
- 注入：`if (n > 0 && n < s.length) n++;`

---

### 第 5 步：SOLO -> IDE 恢复链路
目的：避免强制错误恢复 activitybar 状态。

实现点：SOLO -> IDE 切换路径中的 `nc` 调用。

必须满足：
- 把 `this.nc(!1)` 改为 `this.nc(this.zb.getRuntimeValue(Zt.ACTIVITYBAR_HIDDEN, !0))`

---

### 第 6 步：侧边栏内容高度扣减（布局核心）
目的：活动栏嵌入到侧边栏顶部/底部后，防止侧边栏内容高度叠加，确保 `.content` 与 `split-view-view` 的 `top/height` 正确。

实现点：布局尺寸计算函数 `ZSn.layout(e, t)` 的 `contentSize` 高度表达式。

必须满足：
- 将 `contentSize.height` 从  
`a - s.height - (this.e.verticalPadding ?? 0) * 2`  
调整为  
`a - s.height - n.height - r.height - (this.e.verticalPadding ?? 0) * 2`
- 其中 `n/r` 分别为 `header/footer` 高度，嵌入活动栏时通过该链路自动扣减。
- 禁止在 `create(e)` 或其他创建阶段通过 `style.top/style.height` 直接修补布局。
- 禁止新增临时 DOM 查询与运行时样式覆盖逻辑（例如 `querySelector(...).style...`）。
